两个文件（`agent1.py` 和 `agent.py`）都是用于构建模拟面试官 Agent，但在**实现方式、功能范围、技术架构**等方面存在显著差异，以下是详细对比：

### 一、核心架构差异

表格

|    维度    |                          agent1.py                           |                           agent.py                           |
| :--------: | :----------------------------------------------------------: | :----------------------------------------------------------: |
|  核心框架  |               使用 `langgraph` 手动构建工作流                |     使用 `langchain.agents.create_agent` 快速构建智能体      |
| 工作流设计 | 自定义 `StateGraph`，仅包含单个 `chat` 节点，流程固定（输入→对话→结束） |  基于 LangChain 内置 Agent 框架，自动整合工具调用、对话管理  |
|  状态管理  |        手动定义 `chat_node` 处理消息，手动编译 Graph         | 依赖 `create_agent` 内置的状态管理，结合 `checkpointer` 自动管理 |

### 二、功能范围差异

表格

|    功能点    |             agent1.py              |                          agent.py                           |
| :----------: | :--------------------------------: | :---------------------------------------------------------: |
|   基础对话   |     ✅ 支持（仅纯文本多轮对话）     |                ✅ 支持（作为 Agent 核心能力）                |
|   工具调用   |          ❌ 无任何工具集成          |          ✅ 集成 7 类工具（语音、简历、评估报告等）          |
|   语音对话   |              ❌ 不支持              |     ✅ 集成 `text_to_speech`/`speech_to_text` 等语音工具     |
|   简历分析   |              ❌ 不支持              |  ✅ 集成 `analyze_resume_text`/`extract_resume_highlights`   |
| 面试评估报告 |              ❌ 不支持              | ✅ 集成 `generate_interview_report`/`export_report_markdown` |
|    扩展性    | 需手动修改 Graph 节点 / 边扩展功能 |             只需在 `TOOLS` 列表添加工具即可扩展             |

#### 2. Agent 构建逻辑

- ```
  agent1.py
  ```

  ：手动构建 Prompt → 构建 LLM 链 → 定义 Graph 节点 → 编译 Graph，最终返回 

  ```
  langgraph
  ```

   编译后的图对象：

  python

  

  运行

  ```
  # 手动创建Prompt链
  prompt = ChatPromptTemplate.from_messages([...])
  chain = prompt | llm
  # 手动定义节点和工作流
  def chat_node(state):
      response = chain.invoke({"messages": messages})
      return {"messages": [response]}
  graph_builder = StateGraph(MessagesState)
  graph_builder.add_node("chat", chat_node)
  graph = graph_builder.compile(...)
  # 最终返回graph对象
  ```

  

- ```
  agent.py
  ```

  ：借助 

  ```
  create_agent
  ```

   封装函数，一键整合 LLM、系统提示、工具、状态管理，最终返回 LangChain 标准 Agent 对象：

  python

  

  运行

  

  

  

  ```
  return create_agent(
      model=llm,
      system_prompt=cfg.get("sp"),
      tools=TOOLS,  # 集成工具列表

  )
  ```

  

#### 3. 工具集成

- `agent1.py` 无任何工具列表定义，也无工具调用逻辑；
- `agent.py` 显式定义 `TOOLS` 列表，包含语音、简历、评估报告相关工具，且 `create_agent` 会自动处理工具的选择与调用。

### 四、适用场景差异

- **agent1.py**：适用于**极简的纯文本对话场景**，仅需要基础的多轮对话能力，无复杂工具调用需求，适合快速验证对话流程；
- **agent.py**：适用于**完整的模拟面试场景**，需要整合语音交互、简历分析、面试报告等复杂功能，具备生产级的扩展性和工具调用能力。

### 总结

`agent1.py` 是**基础版**，仅实现了 LangGraph 驱动的纯文本对话；`agent.py` 是**增强版**，基于 LangChain Agent 框架整合了各类业务工具，功能更完整、扩展性更强，是面向实际业务场景的实现。
