### 统大模型调用（直接问答）

```plaintext
用户问题 → 直接调用 LLM → 返回答案
```

**特点**：

- ❌ 无状态 - 每次对话独立，不记得之前说了什么
- ❌ 无工具 - 只能回答，无法执行操作（查数据库、发邮件、搜索）
- ❌ 无流程 - 不会按照预定的面试流程走
- ❌ 无记忆 - 无法保存面试记录、生成报告

------

### Agent 架构（我们的系统）

```plaintext
用户问题 → Agent 调度 → 工具链(可选) → LLM → 响应
                    ↓
              对话记忆存储
                    ↓
              数据库持久化
```

**特点**：

- ✅ **有状态** - 通过 Memory Saver 实现多轮对话记忆
- ✅ **有工具** - 可以调用 ASR/TTS、简历解析、报告生成、数据库
- ✅ **有流程** - 按照面试 SOP 执行（开场 → 追问 → 反馈 → 报告）
- ✅ **可扩展** - 轻松添加新工具、新能力
